# Кристина Беридзе — уроки грузинского языка

Одностраничный русскоязычный сайт преподавателя грузинского языка. Проект построен на React, TypeScript, Vite/vinext, Tailwind CSS, Framer Motion и Lucide React.

## Локальная разработка

Требуется Node.js 22.13 или новее.

```bash
npm install
npm run dev
```

Локальный адрес по умолчанию: `http://localhost:3000/`.

## Проверка и сборка

```bash
npm run lint
npm run build
```

Готовый проект совместим с развёртыванием через OpenAI Sites/Cloudflare Workers. Канонический адрес и SEO-метаданные настроены на `https://kristinalanguages.com/`; перед публикацией на другом домене замените его в метаданных, `robots.txt` и `sitemap.xml`.

## SEO после подключения домена

Канонический адрес, Open Graph, JSON-LD, `robots.txt`, sitemap и manifest настроены на `https://kristinalanguages.com/`. После подключения домена:

1. Откройте [Google Search Console](https://search.google.com/search-console/).
2. Добавьте ресурс типа **Домен** для `kristinalanguages.com`.
3. Скопируйте выданную Google TXT-запись в DNS домена и дождитесь подтверждения.
4. В разделе **Файлы Sitemap** отправьте `https://kristinalanguages.com/sitemap.xml`.
5. В **Проверке URL** проверьте `https://kristinalanguages.com/` и запросите индексирование после того, как домен начнёт открывать этот сайт.

Аналитика и рекламные пиксели намеренно не установлены. При необходимости позже можно подключить Google Search Console без cookie-баннера; для аналитики предпочтителен приватный сервис без межсайтового отслеживания либо отдельная проверка требований к согласию пользователей.

## Активация формы — обязательно один раз

Форма отправляет заявки без перезагрузки через AJAX FormSubmit на `K.beridze1982@gmail.com`. После самой первой реальной отправки FormSubmit пришлёт на этот адрес письмо активации. Кристине необходимо открыть письмо и подтвердить форму один раз; до подтверждения последующие заявки не будут доставляться как обычные письма.

Активация зависит от доступа к почтовому ящику и не подтверждалась в рамках локальной проверки проекта. После активации рекомендуется отправить одну тестовую заявку с опубликованного сайта и убедиться, что письмо пришло. Форма включает обязательные поля, клиентскую валидацию, состояние отправки, сообщения об успехе и ошибке, honeypot и ссылку на WhatsApp на случай сбоя.
